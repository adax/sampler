TokenList component offers a simplified way of working with lists: instance names are listed vertically or horizontally, adding is done using drop-down list, removal - using the buttons located near each instance.

In this example, we have defined two collection data sources: orderItemsDs nested datasource, which includes a collection of order lines within an order, and allItemsDs datasource containing a collection of all order lines available in the database. The TokenList component displays the content of the orderItemsDs datasource and allows changing the collection by adding instances from allItemsDs.

tokenList elements:

  • lookup - values selection component descriptor.
  • button - descriptor of the button for adding items. Can contain caption and icon attributes.
  • clearButton - descriptor of the button for removing items. Can contain caption and icon attributes.
