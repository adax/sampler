lazy attribute sets lazy loading for a tab content.

Lazy-tabs do not load their content when a screen is opened, it reduces the number of components in memory. Components within a tab are loaded only when user selects the tab. Additionally, if a lazy-tab includes visual components linked to a data source, containing a JPQL query, this query will not be invoked as well. As a result, the screen opens quicker and data is loaded only when user requests it by selecting this tab.

Please note that the components located on a lazy-tab do not exist when the screen is opened. That is why they cannot be injected into a controller and cannot be obtained by invoking getComponent() in the controller's init()method. The lazy-tab components can be called only after user opens it. This moment may be caught using Accordion.TabChangeListener.
